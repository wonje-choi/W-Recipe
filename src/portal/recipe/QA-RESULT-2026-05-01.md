# Recipe Taste Optimizer Integration QA Result

- **Date**: 2026-05-01
- **Task**: FN-20260501-0044
- **Scope**: 테스트 데이터 기반 MVP 통합 QA
- **Project**: `main`

## Summary

테스트용 SQLite DB(`data/recipe.db`)에 admin/user 계정, 승인/검수대기/숨김 레시피, 버전, AI 로그, 출처, 신고/수정요청 데이터를 시드한 뒤 공개 API, 사용자 API, 관리자 API, AI 개량, 보안 실패 케이스를 검증했다.

최종 스모크 기준 핵심 MVP 흐름은 통과했다.

## Seed Data

- `qa_admin`: 관리자 계정
- `qa_user`: 일반 사용자 계정
- `qa_dish_low` / `qa_ver_low`: 승인된 저염식 레시피
- `qa_dish_baby` / `qa_ver_baby`: 승인된 이유식 레시피
- `qa_dish_pending` / `qa_ver_pending`: 검수 후 공개 확인, 이후 숨김 처리 확인
- `qa_dish_hidden`: 비공개 확인용 숨김 레시피
- `qa_source_failed`: 실패한 외부 출처
- `qa_ai_pending`: 관리자 반려 확인용 AI 결과

## Issues Found And Fixed

| Issue | Fix | Verification |
| --- | --- | --- |

| 넓은 admin detail route가 `/api/admin/recipes` 목록 경로를 먼저 잡음 | `recipe-admin-detail`에 list/create fallback 추가 | 관리자 레시피 목록 200, total=4 |
| 넓은 admin source detail route가 `/api/admin/sources` 목록 경로를 먼저 잡음 | `recipe-admin-source-detail`에 list/create fallback 추가 | 실패 출처 목록 200, total=1 |
| 넓은 AI prompt detail route가 `/api/admin/ai/prompts` 생성 경로를 먼저 잡음 | `recipe-admin-ai-prompt-detail`에 list/create fallback 추가 | prompt 생성 201, 목록 200 |

## Review Flow

- `GET /api/admin/reviews/pending` returned pending dish/version/AI counts.
- `qa_dish_pending` and `qa_ver_pending` were approved and became publicly readable.
- `qa_ai_pending` was rejected and stayed non-public.
- `qa_dish_pending` was hidden after approval and became publicly unreadable again.
- Admin logs were created for review and hide actions.

## Notes

- Source info and view count endpoints use the documented plural paths: `/api/recipes/<dish_id>/versions/<version_id>/source` and `/api/recipes/<dish_id>/views`.
- AI user requests require active prompt versions for the target purpose. QA created active `low_sodium` and `baby_food` prompt versions through the admin prompt API.
- WIZ server was not restarted; changes were applied through normal build and hot reload.
