# Recipe Taste Optimizer MVP Scope and Information Architecture

- **작성일**: 2026-05-01
- **작업 ID**: FN-20260501-0001
- **목적**: 긴 서비스 요구사항을 실제 개발 가능한 MVP 범위, 1차 정보구조, 공통 상태값 기준으로 고정한다.

## 1. MVP 목표

Recipe Taste Optimizer의 MVP는 “관리자 검수 기반의 한국어 레시피 조회/관리 + AI 요약/개량 + 저염식/이유식 안전 표시”를 검증하는 데 집중한다.

핵심 가치는 다음 문장으로 고정한다.

> 일반 레시피를 더 건강하게, 저염식은 더 맛있게, 이유식은 더 안전하고 쉽게. AI가 정리하고 관리자가 검수하는 레시피 플랫폼.

MVP는 외부 콘텐츠를 대체하는 서비스가 아니라, 원본 출처를 명확히 안내하고 AI가 요약/재구성/개량한 정보를 관리자 검수 후 제공하는 플랫폼으로 설계한다.

## 2. MVP 포함 범위

| 영역 | 포함 기능 | 구현 기준 |
|---|---|---|
| 공개 탐색 | 메인, 레시피 목록, 상세, 저염레시피, 이유식레시피 | 비로그인 사용자도 Approved 레시피 조회 가능 |
| 레시피 구조 | 요리 단위와 레시피 버전 분리 | 하나의 요리에 원본 요약/AI 저염/AI 이유식/AI 간편식 버전 연결 |
| 출처 표시 | 원본 제목, 작성자, URL, 출처 유형, 수집일 | 상세 하단에 원본 이동 버튼 제공, 전문 공개 금지 |
| 관리자 검수 | Draft/Pending Review/Approved/Rejected/Hidden 상태 관리 | AI 결과와 외부 수집 결과는 승인 전 공개 금지 |
| AI 요약 | 외부 자료 또는 직접 입력 텍스트를 레시피 구조로 정리 | 재료, 계량, 순서, 팁, 불확실 정보 분리 |
| AI 개량 | 저염식, 이유식, 맛 개선, 재료 단순화, 조리시간 단축 | 결과는 Pending Review 저장 후 관리자 승인 필요 |
| 안전 표시 | 저염식 주의, 이유식 주의, 알레르기/위험 재료 표시 | 의료 조언/치료 보장 표현 금지 |
| 사용자 참여 | 로그인, 댓글, 즐겨찾기, 수정 요청, AI 개량 요청, 마이페이지 | user role 이상만 쓰기 기능 가능 |
| 관리자 운영 | 레시피 관리, AI 결과 검수, 수집 출처 관리, 댓글/신고/수정 요청 관리, 로그 확인 | 모든 관리자 API는 backend role 검증 필수 |
| QA/운영 | 안정성, 보안, 운영 정책, 배포 체크리스트 | MVP 출시 전 체크리스트로 검증 |

| 다국어 서비스 | MVP는 한국어 품질과 안전 문구 안정화가 우선이다. | Phase 5+ |
| 네이티브 모바일 앱 | 반응형 웹으로 먼저 사용성을 검증한다. | Phase 5+ |
| AI 결과 자동 공개 | 건강/이유식/알레르기 위험 때문에 검수 원칙과 충돌한다. | 제외 |
| AI 의료 상담 표현 | 서비스 범위는 일반 요리 정보이며 진단/치료 표현은 금지한다. | 제외 |

## 5. 1차 메뉴 구조

공개 서비스는 별도 로그인 페이지 없이 메인페이지로 진입한다.

| 메뉴 | 경로 후보 | 접근 권한 | 설명 |
|---|---|---|---|
| 메인페이지 | `/` 또는 `/dashboard` | guest 이상 | 검색, 인기/랜덤/최신 레시피, 특수식단 소개 |
| 레시피 | `/recipes` | guest 이상 | 전체 레시피 목록, 검색, 필터, 정렬 |
| 저염레시피 | `/low-sodium` | guest 이상 | 저염식 레시피와 나트륨 개선 포인트 |
| 이유식레시피 | `/baby-food` | guest 이상 | 월령/단계별 이유식과 안전 필터 |
| 마이페이지 | `/my` | user 이상 | 프로필, 선호도, 활동 내역 |
| Admin Dashboard | `/admin` | admin | 관리자 대시보드와 운영 메뉴 |

로그인 버튼은 상단 또는 좌측 메뉴의 우측/하단에 배치한다. 버튼 클릭 시 일반 로그인/관리자 로그인 탭이 있는 모달을 연다. UI 탭은 분리되어도 실제 인증/인가 구조는 하나의 role 기반 시스템을 사용한다.

## 6. 화면별 1차 정보구조

### 6.1 Main Page

- Hero 문구, 검색창, 추천 키워드
- AI 개량 요청 버튼, 저염식/이유식 바로가기 버튼
- 인기 레시피 3개
- 랜덤 레시피 2개
- 특수식단 개선 소개와 원본/AI 비교 예시
- 최신 업데이트 레시피

### 6.2 Recipe List Page

- 검색창
- 카테고리 필터
- 태그 필터
- 정렬: 조회수순, 최신순, 인기순, 난이도순, 조리시간순, AI 개량순
- 레시피 카드 목록
- 결과 없음, 이미지 없음, 승인 레시피 없음, 필터 결과 없음 상태

### 6.3 Recipe Detail Page

- 요리 기본 정보
- 같은 요리의 레시피 버전 목록
- 선택 버전 상세: 재료, 계량, 조리 순서, 팁, 영양/나트륨/알레르기 정보
- 원본/AI 개량 비교 표
- AI 개량 레시피 영역
- 원본 출처 링크 영역
- 댓글 영역
- 수정 요청 영역

### 6.4 Low Sodium Page

- 저염식 레시피 목록
- 고나트륨 재료 표시
- 저염 개량 포인트
- 예상 나트륨 감소 포인트
- 맛 보완 재료 추천
- 건강 관련 주의 문구

### 6.5 Baby Food Page

- 초기/중기/후기/완료기 이유식 필터
- 권장 월령, 입자 크기, 보관 방법
- 알레르기/질식 위험/꿀/견과류/생식 재료 경고
- 보호자 확인과 전문가 상담 안내 문구

### 6.6 My Page

- 프로필 정보
- 저장한 레시피
- 최근 본 레시피
- 댓글 내역
- 수정 요청 내역
- AI 개량 요청 내역
- 관심 카테고리, 알레르기, 비선호 재료, 선호 식단, 조리 가능 시간, 보유 조리도구 설정

### 6.7 Admin Dashboard

관리자 페이지는 일반 사용자 화면과 분리된 대시보드로 구성한다.

- Dashboard
- Recipe Management
- User Management
- Comment & Report Management
- AI Management
- Crawled Source Management
- Edit Request Management
- System Log

## 7. 공통 상태값 초안

### 7.1 User Role

| 값 | 설명 |
|---|---|
| `guest` | 비로그인 사용자. 공개 조회만 가능 |
| `user` | 일반 로그인 사용자. 참여 기능 가능 |
| `admin` | 관리자. 모든 관리자 API 접근 가능 |

### 7.2 User Status

| 값 | 설명 |
|---|---|
| `Active` | 정상 계정 |
| `Suspended` | 일시 정지 계정 |
| `Deleted` | 탈퇴 또는 비활성 계정 |
| `Pending` | 이메일 인증 등 대기 상태가 필요한 경우 |

### 7.3 Recipe Status

| 값 | 설명 | 공개 여부 |
|---|---|---:|
| `Draft` | 관리자가 작성 중 | 비공개 |
| `Crawled` | 외부 자료 수집 완료 | 비공개 |
| `AI Parsed` | AI 요약/정리 완료 | 비공개 |
| `AI Modified` | AI 개량 완료 | 비공개 |
| `Pending Review` | 관리자 검수 대기 | 비공개 |
| `Approved` | 관리자 승인 완료 | 공개 |
| `Rejected` | 반려 | 비공개 |
| `Hidden` | 운영상 숨김 | 비공개 |

### 7.4 AI Processing Status

| 값 | 설명 |
|---|---|
| `Queued` | 처리 대기 |
| `Processing` | AI 처리 중 |
| `Pending Review` | 결과 생성 후 관리자 검수 대기 |
| `Approved` | 관리자 승인 |
| `Rejected` | 관리자 반려 |
| `Failed` | 처리 실패 |

### 7.5 Source Type

| 값 | 설명 |
|---|---|
| `YouTube` | 유튜브 영상/채널 기반 자료 |
| `Blog` | 블로그 기반 자료 |
| `Web` | 일반 웹문서 기반 자료 |
| `Direct` | 관리자 또는 사용자가 직접 입력한 텍스트 |
| `AI Modified` | AI가 기존 레시피를 개량한 버전 |

### 7.6 Comment Status

| 값 | 설명 |
|---|---|
| `Visible` | 공개 댓글 |
| `Hidden` | 관리자 숨김 댓글 |
| `Deleted` | 삭제 처리 댓글 |
| `Reported` | 신고 접수 댓글 |

### 7.7 Edit Request Status

| 값 | 설명 |
|---|---|
| `Open` | 접수됨 |
| `In Review` | 검토 중 |
| `Resolved` | 처리 완료 |
| `Rejected` | 반려 |

### 7.8 Report Status

| 값 | 설명 |
|---|---|
| `Open` | 신고 접수 |
| `In Review` | 관리자 검토 중 |
| `Actioned` | 조치 완료 |
| `Rejected` | 신고 반려 |

### 7.9 Crawling Status

| 값 | 설명 |
|---|---|
| `Pending` | 수집 대기 |
| `Allowed` | 수집 가능 확인 |
| `Blocked` | robots.txt/약관/차단 도메인 등으로 수집 제한 |

## 9. 필수 운영 원칙

- 외부 원문 전문을 사용자에게 그대로 노출하지 않는다.
- 원본 제목, 작성자, URL, 출처 유형, 수집일을 저장한다.
- 상세 하단에 원본으로 이동할 수 있는 출처 링크를 제공한다.
- AI 결과는 요약, 재구성, 개량 정보로 제공한다.
- AI가 생성하거나 크게 변경한 레시피는 바로 공개하지 않는다.
- 저염식, 이유식, 건강 관련 식단, 알레르기 포함 콘텐츠는 관리자 검수 후 공개한다.
- 건강상 제한이 있는 사용자는 전문가 상담이 필요하다는 안내 문구를 제공한다.
- 의료적 진단, 치료, 질병 개선을 보장하는 표현은 금지한다.
