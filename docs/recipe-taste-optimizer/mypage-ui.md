# 마이페이지 UI

`page.mypage`는 Recipe 앱의 로그인 사용자 마이페이지다. 기존 일반 사이드바 화면을 `layout.recipe` 기반 화면으로 교체하고 `/mypage` 경로를 유지한다.

## 경로

| 경로 | 설명 |
| --- | --- |
| `/mypage` | 프로필, 활동 내역, 개인화 설정 화면 |

## API 함수

| 함수 | 설명 |
| --- | --- |
| `load()` | 프로필, 선호도, 활동 내역, 옵션 목록 반환 |
| `save_profile()` | 닉네임 저장 |
| `save_preference()` | 개인화 설정 저장 |

`page.mypage`는 `user` controller를 사용하므로 로그인 사용자가 아니면 접근할 수 없다.

## UI 구성

### 프로필

- 이메일, 역할 읽기 전용 표시
- 닉네임 수정

### 활동 내역

- 저장한 레시피
- 최근 본 레시피
- 댓글 내역
- 수정 요청 내역
- AI 개량 요청 내역

저장한 레시피와 최근 본 레시피는 공개 Version이 있으면 `/recipes/detail/{dishId}?version={versionId}`로 이동한다.

### 개인화 설정

- 관심 식단
- 알레르기
- 비선호 재료
- 선호 식단
- 선호 조리 시간
- 조리도구
- 아기 월령
- 나트륨 선호
- 식감 선호

## 데이터 정책

- 프로필 응답은 `struct.user.safe_profile()`을 사용해 password hash, 로그인 실패 횟수, 잠금 정보 등 내부 필드를 제외한다.
- 선호도 배열 값은 UI에서 쉼표 입력 또는 토글 값으로 편집하고, 저장 시 JSON 문자열로 직렬화된다.
- 활동 내역은 현재 로그인 사용자 자신의 데이터만 조회한다.
