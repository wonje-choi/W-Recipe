# 관리자 참여/신고/수정 요청 관리 UI

`page.admin.feedback`는 댓글, 신고, 수정 요청을 운영자가 검토하고 처리하는 관리자 화면이다.

## 경로

| 경로 | 설명 |
| --- | --- |
| `/admin/feedback` | 관리자 참여 관리 댓글 탭 |
| `/admin/feedback?tab=reports` | 신고 관리 탭 |
| `/admin/feedback?tab=edit-requests` | 수정 요청 관리 탭 |

## 화면 구성

| 탭 | 설명 |
| --- | --- |
| 댓글 | 전체 댓글, 신고된 댓글 필터, 상태 필터, 댓글 숨김/삭제/복구, 사용자 제재 |
| 신고 | 신고 사유, 대상 유형, 대상 댓글/레시피, 처리 상태, 관리자 메모 관리 |
| 수정 요청 | 요청 유형, 요청자, 대상 레시피, 요청 내용, 처리 상태, 관리자 메모 관리 |

## 앱 API

| 함수 | 설명 |
| --- | --- |
| `options()` | 댓글/신고/수정 요청/사용자 상태 옵션 반환 |
| `comments()` | 댓글 목록과 신고 댓글 필터 결과 반환 |
| `reports()` | 신고 목록 반환 |
| `edit_requests()` | 수정 요청 목록 반환 |
| `comment_action()` | 댓글 숨김, 삭제, 표시 복구 처리 |
| `report_action()` | 신고 상태를 검토 중, 조치 완료, 반려로 변경 |
| `edit_request_action()` | 수정 요청 상태를 검토 중, 해결, 반려로 변경 |
| `user_action()` | 사용자 상태를 정지 등으로 변경 |

## 감사 로그

다음 작업은 모두 `admin_action_log`에 before/after 값을 남긴다.

- 댓글 숨김, 삭제, 표시 복구
- 신고 상태 변경
- 수정 요청 상태 변경
- 사용자 상태 변경

## 권한

`page.admin.feedback`는 `admin` controller를 사용한다. 일반 사용자는 화면과 API를 사용할 수 없다.
